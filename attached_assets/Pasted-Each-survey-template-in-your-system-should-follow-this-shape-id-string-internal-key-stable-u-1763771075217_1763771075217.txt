Each survey template in your system should follow this shape:

id (string) – internal key, stable, used in code/DB

name (string) – human label shown in UI

description (string) – short explanation for admins/trainers

timing (enum) – e.g. "before_training" | "during_training" | "immediately_after_session" | "immediately_after_training" | "weeks_after_training" | "weeks_or_months_after_training"

audience (enum/string) – e.g. "participants" | "managers" | "participants_or_managers"

questions (array of question objects)

Each question:

id (string) – unique per survey

text (string) – question text

type (enum) – "single_select" | "multi_select" | "rating" | "text" | "number"

required (boolean)

optional:

options (array of { value, label }) for select types

scale for rating: { min, max, labels: { [key:string]: string } }

help_text (string)

You can literally drop this into surveys.json in Replit and JSON.parse it.
